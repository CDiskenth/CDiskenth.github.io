前言

Linux經常作為軟路由的系統使用，而普通的路由器通常有NAT模式、橋接模式等等。當我們將其作為網橋使用時，每一個接入Linux軟路由的主機皆處於同一個網段，很容易互相訪問。

本文描述的是橋接模式的配置。

目標

通過iproute2，systemd-networkd，使家用PC接至軟路由後，能夠通過其訪問網路，且PC機與軟路由處於同一網段

測試環境

  • 一台家用PC
  • Linux軟路由
  • 一台家用路由器

軟路由使用Nano Pi R5S，家用PC為一般的Windows主機，兩台主機都連到同一台家用路由器。各設備參數如下：

家用PC：

參數
Architecture x86
CPU Intel i9-9880H
System Windows
IP 192.168.8.173

Nano Pi R5S：

參數
Architecture ARM
CPU Rockchip RK3568
System Arch linux ARM
User zeta
IP 192.168.8.215

家用路由器：

參數
IP 192.168.8.1

網路拓樸

軟路由網卡名稱與網口的對應關係為：

網卡名稱 網口
end0 WAN口
end1 LAN1
enP1p17s0 LAN2

其中，

  1. WAN口連接至家用路由器的LAN口
  2. LAN1口連接至Windows主機
  3. LAN2口不使用

添加主網路至網橋

  1. 查看目前end0(即WAN口)的網絡信息

可以得知當前軟路由的外網信息為：

  • WAN口IP：192.168.8.215
  • 默認網關：192.168.8.1
  • 子網掩碼：255.255.255.0(即網域”192.168.8.0/24”中的24)
  1. 添加外網信息至br2
  1. 為br2添加路由表

這裡預先給br2添加了一個默認網關192.168.8.1(即家用路由器的IP地址)，以防止後續將當前網卡end0切換成br2時造成網路斷線

  1. 將主網路添加至br2，並移除原end0的IP地址

執行這兩條指令的時間間隔盡量短，因為將主網路end0添加到網橋br2後會造成兩個設備同時具有相同的IP，這樣容易讓已連線中的網路服務斷線

基於systemd-networkd的持久化配置

systemd-networkd是用於配置網路參數的包<span class=”hint–top hint–error hint–medium hint–rounded hint–bounce” aria-label=”systemd-networkd - ArchWiki“>2，默認已經包含在系統內置的Systemd中，不須另外安裝

由於使用iproute2配置的網橋僅能暫時生效，每次系統重新啟動後都必須重新配置，故更常用的配置方式是使用Linux內置的systemd-networkd包來實現

配置文件

參考Arch Linux Wiki<span class=”hint–top hint–error hint–medium hint–rounded hint–bounce” aria-label=”systemd-networkd - ArchWiki“>2，主要有以下配置文件及目錄：

  • /etc/systemd/networkd.conf: 全局配置文件
  • /etc/systemd/network/: 系統管理員所建立的配置文件(優先級較高)
  • /usr/lib/systemd/network/: 新安裝完成後，個別網卡默認的配置文件目錄(優先級較低)

配置文件類型主要有：

  • .network: 用於對應網卡的配置文件
  • .netdev: 用於創建虛擬網卡

這裡應該要將配置文件放到/etc/systemd/network/目錄下，而/usr/lib/systemd/network/目錄下的文件不更動
