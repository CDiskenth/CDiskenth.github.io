NanoPi R5S軟路由基於iptables的NAT服務器

目標

本文演示如何配置網卡及iptables的NAT功能，使接入Linux軟路由的家用PC能夠上網，且家用PC與其它設備不同，屬於另一個子網域

測試環境

  • 一台家用PC
  • Linux軟路由
  • 一台家用路由器

軟路由使用Nano Pi R5S，家用PC為一般的Windows主機，兩台主機都連到同一台家用路由器。各設備參數如下：

家用PC：

參數
Architecture x86
CPU Intel i9-9880H
System Windows
IP 192.168.8.173

Nano Pi R5S：

參數
Architecture ARM
CPU Rockchip RK3568
System Arch linux ARM
User zeta
IP 192.168.8.215

家用路由器：

參數
IP 192.168.8.1

網路拓樸

軟路由網卡名稱與網口的對應關係為：

網卡名稱 網口
end0 WAN口
end1 LAN1
enP1p17s0 LAN2

其中，

  1. WAN口連接至家用路由器的LAN口
  2. LAN1不使用
  3. LAN2口接至家用PC

＊注意，在軟路由中這三個網口已經配置為網橋br2

配置

配置網卡

目前網卡end0、end1、enP1p17s0都屬於br2這個網橋，這裡將enP1p17s0這張網卡從網橋中移除，其餘則不更動

接著給其添加IP位址192.168.7.1/24

啟用內核IPv4轉發

重新指定PC機網卡為靜態IP，位址為192.168.7.3，子網掩碼為255.255.255.0，默認網關指定為192.168.7.1(即軟路由的IP位址)，並關閉防火牆

＊關於Windows操作系統如何更改IP為址，具體操作請Google相關資料

連接PC機至軟路由的enP1p17s0網卡(LAN2網口)，確認PC機是否能ping通軟路由

可以看出PC機可以ping通軟路由。

接著測試軟路由是否能ping通PC機

軟路由亦可ping通PC機，至此網卡的配置完成

配置iptables

使用的iptables版本為v1.18.11-2

iptables配置文件

相關的配置文件為：

  • /etc/iptables/iptables.rules：主要的配置文件，紀錄防火牆的規則
  • /etc/iptables/empty.rules：最原始的配置，想恢復默認值時可參考此文件

撰寫防火牆規則

  1. 查看iptables防火牆目前的規則

所有表、鏈的目標(target)皆為ACCEPT(即放行所有封包)

  1. 若已經有先前制定的規則存在，可執行以下命令將其全部刪除，若沒有則跳至下一步

＊注意此步驟不會清除iptables防火牆的默認規則

  1. 放行與特定服務相關的封包進入主機
  • -t filter指定為filter表，此表用於目標為Linux本地的封包
  • -i br2表示欲配置的設備為網橋，但也能是網卡(如eth0、end0等…)，因此這裡需要根據主機所用的網路設備來修改
  • --dport 22表示封包的目標端口號，這裡指定為SSH的端口(即22)
  • -j ACCEPT表示目標(動作)為放行，即允許通過防火牆

由於SSH和ICMP協議經常使用，所以允許通過防火牆
＊ICMP為網路層協議，和TCP、UDP屬於同一層級，並不具備端口號，因此不使用--dport參數

  1. 開啟IP偽裝**(重點)**

這一步讓網卡enP1p17s0所在的網域(192.168.7.0/24)中的所有傳進來的封包的來源IP被替換成網卡br2的IP（即把其他主機的真實IP偽裝成192.168.8.215）

  1. 保存配置
    用iptalbes-save配合重定向來將配置寫入/etc/iptables/iptables.rules
  1. 啟動iptables服務並配置持久化
  1. 更改默認政策後再次保存配置

默認狀態下，把欲進入防火牆且目標為Linux本地的封包全部丟棄，但目標不是Linux本機或是欲從本地出去防火牆的封包予以放行

＊在最後一步才配置默認政策的原因為不論iptables是否已經啟動，sudo iptables -t filter -P INPUT DROP這項默認政策在設定時都會造成所有進入主機的封包被丟棄，通過ssh遠程登入的使用者必然會斷線，且再也無法重新登入

配置內核IPv4封包轉發

為了實現ipv4封包轉發需要配置內核參數並且使配置永久生效，Linux內置的sysctl已經提供了相關功能

Arch Linux中sysctl會按順序採用下列目錄的配置文件：

  • /etc/sysctl.d/*.conf(優先級最高)
  • /run/sysctl.d/*.conf
  • /usr/local/lib/sysctl.d/*.conf
  • /usr/lib/sysctl.d/*.conf
  • /lib/sysctl.d/*.conf(優先級最低)
    ＊一旦高優先級目錄下的文件被採用，那其餘低優先級目錄下的同名文件會直接忽略
